== Introduction and goals

=== Requirements

The purpose of this project is to develop a full-stack web application for a game named YOVI, communicating human players and automated agents. The following specifications must be in the final version of the aplication:

==== User Management
[cols="1,4"]
|===
| ID | Description
| FR-01
| The system must allow users to register and authenticate (login/logout) to manage their personal  profiles.
| FR-02
| The system must maintain a persistent history of matches played by each registered user.
| FR-03
| The system must provide a profile for users to view their statistics, including win/loss ratios and total games played.
|===

==== Gameplay (Web Frontend)
[cols="1,4"]
|===
| ID | Description
| FR-04
| The frontend shall implement the "Classic Version" of Game Y in a Player-vs-Machine (AI) mode.
| FR-05
| The system must support variable board sizes, allowing the player to define the dimensions before starting the game.
| FR-06
| The UI should allow the player to select from multiple game strategies or difficulty levels provided by the AI.
| FR-07
| The frontend shall visualize the board using the coordinate systems (Barycentric or Index-based) defined in the YEN notation.
|===
